Input

Process

반려·감액 대응 검토 — CASE_004는 생성 대상 없음

denial-response Phase 2 · 약관 매칭/반박 포인트

④ 반려·감액 대응 검토
Input

denial_reason_result.json, normalized_policy_clause.json, requirement_matching_result.json

Process

1. 감액사유·약관 입력 확인: denial_reason_result.json과 normalized_policy_clause.json의 비어 있음 여부를 먼저 확인.
2. Policy-to-Denial Matching: 감액사유가 있으면 관련 약관 조항과 연결하고, 반박 가능한 쟁점을 찾음.
3. 선제 검토 플래그 유지: 음주, 가입금액, 치료종결 시점처럼 확인이 필요한 쟁점은 반박이 아니라 review-only로 남김.
4. Rebuttal Point Generation: 원문 근거가 있는 경우에만 반박 포인트와 권고 문구를 생성.
5. Markdown 요약: 사람 검토용 rebuttal_points.md로 정리하되, CASE_004는 실제 감액사유가 없어 생성 포인트 0건.

Output
